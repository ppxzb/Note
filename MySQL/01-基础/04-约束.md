## 一、非空约束（NOT NULL）

```text
限制该字段的数据不能为 null
```

## 二、唯一约束（UNIQUE）

```text
保证该字段的所有数据都是唯一、不重复的
```

## 三、主键约束（PRIMARY KEY）

```text
主键是一行数据的唯一标识，要求非空且唯一
```

> 配合 `AUTO_INCREMENT` 自动增长

## 四、默认约束（DEFAULT）

```text
保存数据时，如果未指定该字段值，则采用默认值
```

## 五、检查约束（CHECK）

```text
保证字段值满足某一个条件
```

## 六、外键约束（FOREIGN KEY）

```text
用来连接两张表的数据，保证数据的一致性和完整性
```

### 1. 添加外键

- 创建表时添加外键
```sql
CREATE TABLE 表名 (
  ...
  [CONSTRAINT] [外键名称] 
  FOREIGN KEY (外键字段名) REFERENCES 主表(主表列名)
);
```

- 表创建后添加外键
```sql
ALTER TABLE 表名
ADD CONSTRAINT 外键名称
FOREIGN KEY(外键字段名) REFERENCES 主表(主表列名);
```

### 2. 删除外键

```sql
ALTER TABLE 表名 DROP FOREIGN KEY 外键名称;
```

### 3. 外键删除/更新行为

```sql
ALTER TABLE 表名
ADD CONSTRAINT 外键名称
FOREIGN KEY(外键字段名) REFERENCES 主表(主表列名)
ON UPDATE 外键行为
ON DELETE 外键行为;
```

外键行为：
- `NO ACTION`：父表删/改时先检查子表引用，有引用则不允许  
- `RESTRICT`：与 NO ACTION 一致  
- `CASCADE`：父表删/改时同时删除/更新子表记录  
- `SET NULL`：父表删除时子表外键置 NULL  
- `SET DEFAULT`：父表变更时子表外键设默认值（InnoDB 不支持）
